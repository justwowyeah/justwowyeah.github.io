Hexo使用报告

发表于 2023-06-25 分类于 学习 ， 博客 本文字数： 503 阅读时长 ≈ 2 分钟

Hexo使用文档告诉我们：

你可以执行下列命令来创建一篇新文章或者新的页面。

写作 |Hexo

  • 首位的 $ 是 Git Bash 自动显示的，不是你应该输入的内容

因此，当你想要写一篇题为 “第一篇博客” 的草稿时，你可能会尝试：hexo new [draft] <第一篇博客>
但你不会收到任何效果，因为这种做法实际上是错误的。你真正应该敲下的是：hexo new draft 第一篇博客

不仅如此，在后文中你会见到这么一个符号，它频繁出现，但是通常是可以被忽略的：""
这是 “” 在代码框中的样式，它不同寻常，不是吗？

由此，我们得知了第一个重要的规则：

[ ]、<>、""通常不是输入内容，而是提示输入的存在。

在阅读 Hexo使用文档 的时候，这一点是不得不记住的，因为你很可能会不断地栽在这上面。试想：一篇帮助文档的第一句话（实际为基本操作的第一句话）就隐含了如此巨大的未知信息，它显然把真正的初学者看得太聪明了，如果你不幸正欲从此入门，那么阅读本文是你的不二之选。

让我们继续。Hexo默认添加了三种布局模板：post、draft、page,它们全部位于 /scaffolds ，下面一一详谈：

POST

post：发布，Hexo的默认布局。当你使用命令：hexo new 第一篇博客
Hexo会自动生成 /source/_post/第一篇博客/第一篇博客.md
这份md文件就是以 /scaffolds/post.md 为基础制作的，其中的 title 是你生成文件时的命名，date是你创建文件的时间，所有选项你都可以自行更改。当然，不使用命令也是可以的，命令的本质是操作的自动化，越过繁琐且枯燥的过程直接看到结果。
