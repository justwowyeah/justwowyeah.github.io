Markdown语法学习

发表于 2023-07-01 分类于 学习 本文字数： 943 阅读时长 ≈ 3 分钟

写博客是一件怎样的事呢？现在的我面临这个问题。

解铃还须系铃人，答案需得从过去找，来看看近一周我做了什么！

  • 翻看过无数篇建立博客的教程
  • 尝了无数只GitBash告诉我的bug
  • 体验了Hexo配合GithubPages搭建起属于我的个人站点
  • 不断改进ayer主题里的页面布局
  • … …

当这一切逐渐完结，我向自己发出了重要的疑问：接下来怎么办？

答案不必思索，我告诉自己：

向前走，别回头！

《漫长的季节》

让我们先考虑另一个问题：都说万事开头难，现在我渡过了最初的难关了吗？写博客是一件长远的事，如果一物的起源是极重要的，那么一件恒常存在的物，其源头不会简陋。

我还没有走出开头。弄清楚这一点是极重要的。

直到现在，我拢共也就遇到两件事：头一件事是如何搭建起这个网站。现在我解决了这个问题，最初的满腔热情也趋于平静，那么是时候走出下一步了：写文章。可紧接着又遇上一件棘手的事：我怎么写文章呢？

说这句话，其实有两个意思：

  1. 我要说什么？
    关于这一点，现在是没有疑问的，或者说还没到考虑的时候。因为我还没有客人，也就无需烦心内容，即便是记录生活的琐碎嘈杂也没有问题，权当是写日记了。日记又有什么讲究呢？
  2. 我该怎么说？
    实际上，这一条已经成为了亟待解决的事情，同时也是我近来的兴趣所在。

怎么把事情讲出来，这是个问题。让我们忽略语法上的遣词造句，我没必要做到那么精细。

我们只是谈谈技术上的事情：有关 格式 和 排版 。尽管这可能仍然处于语法的范畴，是吗？

我是如何考虑 格式 和 排版 的？毫不意外的，我选择了 Markdown 。

什么是Markdown？

这里我决定引用Markdown官方教程的介绍：

Markdown 是一种轻量级的标记语言，可用于在纯文本文档中添加格式化元素。Markdown 由 John Gruber 于 2004 年创建，如今已成为世界上最受欢迎的标记语言之一。

1.专注于文字内容；
2.纯文本，易读易写，可以方便地纳入版本控制；
3.语法简单，没有什么学习成本，能轻松在码字的同时做出美观大方的排版。

为什么是Markdown？

与预期的答案不同，我并非早早看到了Markdown的优势和效果。不仅没有，我甚至从没有主动了解过Markdown。当然，直到昨天。我对Markdown这一名字的熟悉来自于各种地方，尤其是 readme.md 文件。我相信会有人熟悉我下面所列举的各种场景：

  1. GitHub的几乎每一个项目都能见到它的身影
  2. Magisk的每一个模块都附带了这一说明性文本。
  3. 酷安和互联网其他地方都存在Markdown的讨论区和软件分享。

在一个氛围成型的社区，你有什么办法躲开这些呢？

  • 更重要的是，我使用GithubPages和Hexo建立我的博客，而新建文章的默认后缀名是 .md 。

基于以上原因，我开始了解Markdown语法和纷繁的编辑器。

学习新东西总是很有意思的事情，这一份“意思”一般会很快消亡。短暂的接触告诉我：Markdown可能不是这样。
